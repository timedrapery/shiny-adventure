# Security Policy

## Supported Versions

This repository is actively maintained on the main branch.

Security fixes are made on current code rather than backported release branches.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Use one of the following private channels:

1. GitHub Security Advisories (preferred): use the repository's "Report a vulnerability" flow.
2. If advisory reporting is not available, contact project maintainers directly through GitHub and include "Security" in the subject.

When reporting, include:

- a clear description of the issue
- impact and affected components
- reproduction steps or proof of concept
- suggested remediation if known

## Disclosure Process

Maintainers will:

- acknowledge valid reports as quickly as possible
- investigate and confirm impact
- prepare and publish a fix
- coordinate public disclosure once a fix is available

We appreciate responsible disclosure and will credit reporters when appropriate.
