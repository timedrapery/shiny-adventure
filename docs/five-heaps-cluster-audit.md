# Five Heaps Cluster Audit

## Scope

This audit records the repository state before the March 2026 five-heaps /
clung-to-heaps pass.

The aim is not to prove that the relevant terms already exist somewhere in the
repo. The aim is to identify where the family is already usable, where it still
depends on thin compound stubs, and where neighboring doctrinal clusters are
likely to pull the same terms in different directions.

## 3. High-Value Missing Expressions Or Formula Objects

### Missing headwords

- `sakkāya`

This is the most obvious doctrinal gap in the family because
`sakkāya-diṭṭhi` already exists as a major entry and the heaps cluster depends
on it.

### Missing formula or expression records

- full five-heap enumeration formulas
- `yaṃ kiñci rūpaṃ ... vedanā ... saññā ... saṅkhārā ... viññāṇaṃ` patterns
- heap-context not-self lines from `SN 22.59`
- gratification / danger / escape formulae over the heaps
- burden expressions such as `bhāro have pañcakkhandhā`

The governing headwords mention these environments, but the repo does not yet
present them as reusable translation objects.

## 4. Weak `translation_policy` Or `context_rules`

- `saññā` is structurally complete but still too thin for a family pass. It
  does not yet explain how aggregate use should stay distinct from nearby
  `viññāṇa` language.
- `saṅkhārā` is strong for dependent-arising and three-marks use, but heap
  context is not yet encoded explicitly.
- `viññāṇa` is strong for conditioned-process handling, but it does not yet
  govern heap-context separately from chain-context.
- `upādāna` has strong general policy, but it does not yet govern
  `pañcupādānakkhandhā` as a stabilized expression.
- `khandha` is usable, but it still lacks explicit family-level coordination
  with the five constituent heaps and the `pañcakkhandhā` / `pañcupādānakkhandhā`
  distinction.

## 5. Weak `authority_basis`

The main weakness is not absence but thinness.

The following headwords currently rely on a single authority item:

- `khandha`
- `rūpa`
- `vedanā`
- `saññā`
- `saṅkhārā`
- `viññāṇa`
- `upādāna`
- `sakkāya-diṭṭhi`
- `anattā`

That is enough to pass current validation but weak for a family where
translator decisions are tightly interdependent.

## 7. Highest Drift-Risk Terms

### `saṅkhārā`

This is the sharpest risk in the cluster.

The term now has explicit policy for:

- dependent-arising lines
- three-marks formulae

But it still lacks explicit heap-context policy. That leaves
`saṅkhārakkhandha` exposed to drift back toward `formations`, `volitional
formations`, or other inherited English not explicitly governed by the current
house style.

### `viññāṇa`

This entry already carries:

- six-sense knowing
- dependent-arising knowing
- aggregate use

But aggregate use is not yet strong enough as a separate governed context.

### `upādāna`

The headword now prefers `taking personally`, but the cluster still preserves:

- `five appropriated heaps`
- `five heaps taken personally`
- `five heaps of clinging`

That is already active drift inside one key doctrinal expression.

### `khandha`

The repo has largely stabilized `heap` over `aggregate`, but the compounds and
collective formulas have not yet been updated to behave like one governed
family.

## 8. Cross-Cluster Alignment Needs

### Dependent arising

- `vedanā`
- `saṅkhārā`
- `viññāṇa`
- `upādāna`

These terms now already carry dependent-arising guidance, so the five-heaps
pass must extend rather than contradict that work.

### Self / not-self / self-view

- `anattā`
- `atta`
- `sakkāya-diṭṭhi`
- `attavādupādāna`
- `diṭṭhi`

This is the most important doctrinal overlap after dependent arising.

### Dukkha / origin / escape

- `dukkha`
- `samudaya`
- `nirodha`
- `nissaraṇa`

If the repo adds gratification / danger / escape formulas for the heaps, those
terms need to stay aligned with existing truth-language and liberation
language.

## Audit Summary

The repository already has the right headwords for a five-heaps pass.

The main weaknesses are:

1. the collective expressions and heap-compounds are still thin minor stubs
2. `sakkāya` is missing even though `sakkāya-diṭṭhi` is already major
3. `pañcupādānakkhandhā` already drifts across multiple English renderings
4. `saṅkhārā`, `viññāṇa`, and `upādāna` now need explicit heap-context rules
   so this cluster does not fight the already-strengthened dependent-arising
   cluster

The next pass should therefore:

1. stabilize the headwords as a family
2. promote or strengthen the collective expressions and heap-compounds
3. add formula records for recurring heap formulas
4. add a small cluster report and translator-facing outputs, as was done for
   dependent arising
