# Dependent Arising Cluster Audit

## Scope

This audit records the repository state before the March 2026 dependent-arising
cluster pass.

The purpose is not to restate that the repo already contains the major links.
It is to identify where the family is already usable, where it is still
fragile in live translation work, and where tooling does not yet protect the
cluster from drift.

## 2. Major vs Minor Assessment

### Already functioning as major rule-bearing entries

- `paṭiccasamuppāda`, `taṇhā`, `upādāna`, `vedanā`, `viññāṇa`, `nāmarūpa`,
  `saḷāyatana`, `phassa`, `bhava`, `nirodha`

These entries already have enough policy structure to govern translation work,
but several still rely on a single-source authority basis and do not yet
coordinate strongly enough with the linked formula objects.

### Major entries that exist but still need stronger family-level coordination

- `avijjā`, `saṅkhārā`, `jāti`, `jarāmaraṇa`, `paccaya`, `nidāna`,
  `idappaccayatā`

These entries are structurally complete, but they are not yet fully aligned
with the cluster's formula inheritance, practical here-now framing, or linked
compound behavior.

### Terms that are strategically too lightweight for the current milestone

- all eleven `*-paccaya` link compounds
- all eleven `*-nirodha` link compounds
- `samphassa`

These are still short minor records with little or no rule-bearing guidance,
no examples, no related-term propagation, and no provenance. They are present
in the lexicon but not yet translation-ready as a coordinated system.

## 4. Weak Or Generic `authority_basis`

The main weakness is not empty provenance. It is provenance that is too thin
for a doctrinally sensitive family.

### Cluster anchors relying on a single authority item

- `avijja`
- `sankhara`
- `vinnana`
- `namarupa`
- `salayatana`
- `phassa`
- `vedana`
- `bhava`
- `jati`
- `jaramarana`
- `paccaya`
- `nidana`
- `idappaccayata`

In most of these entries the single supporting item is `OSF glossary` or one
source profile. That is enough to pass current validation, but it is weak for a
 cluster where formula handling and interpretive framing matter.

### Better-shaped entries already showing the desired direction

- `paticcasamuppada`
- `tanha`
- `nirodha`

These already name more than one authority layer or house source and should be
used as the style target for the rest of the cluster.

## 5. Example Citation Gaps

### Major entries

- No current citation gaps were found in major dependent-arising headwords.
- The main issue is not missing citations in existing major examples.
- The main issue is that many high-value compounds and all missing formula
  objects have no examples at all, so translators cannot yet rely on the repo
  for repeated line-level guidance.

### Minor entries

- The forward-link `*-paccaya` minors have no `example_phrases`.
- The reverse-link `*-nirodha` minors have no `example_phrases`.
- `samphassa` has no canonical examples and is still only a draft.

## 6. Wording That Appears Stale Against Current House Style

- The `upādāna` headword now uses `taking personally`, but the four
  `upādāna` compounds still center `clinging` language. Those compounds need
  either explicit override logic or closer inheritance from the headword.
- The taṇhā family is partly stabilized at `ignorant wanting`, but several
  related compounds still preserve older `craving` language as the easiest
  fallback rather than a documented alternate.
- `jāti` is currently centered on `rebirth`, which underuses the repository's
  own Buddhadāsa authority for practical and immediate readings of the term in
  dependent arising.
- `saṅkhārā` and the `X-paccayā Y` lines are still coordinated mainly through
  prose notes, not through dedicated formula records. That leaves the family
  vulnerable to silent line-by-line drift.
- `paṭiccasamuppāda`, `paccaya`, and `idappaccayatā` all point toward the core
  conditionality lines, but the repo does not yet present those lines in one
  stable voice for translators.

## 7. Current Validation Gaps

- No current check reports whether the dependent-arising cluster is complete as
  a family.
- No current check reports whether the standard forward and reverse link
  compounds remain aligned with their governing headwords.
- No current check reports the absence of dedicated formula records for the
  cluster's most reused lines.
- No current check distinguishes strong provenance from thin provenance on
  cluster anchors; a single `OSF glossary` item passes as easily as a fuller
  multi-source basis.
- No current workflow generates a translator-facing cluster glossary, formula
  sheet, or consistency brief from the live data.

## Audit Summary

The repository already contains the right doctrinal family. The problem is not
coverage in the abstract.

The problem is that the dependent-arising family is still split across:

- strong headwords
- thin link compounds
- example-only formula handling
- tooling that checks general structure but not cluster coherence

The next pass should therefore:

1. strengthen the major headwords as one coordinated family
2. upgrade the `*-paccaya` and `*-nirodha` link records from labels to usable
   translation objects
3. create dedicated formula records for the core conditionality and chain lines
4. add a small cluster audit and output generator so translators can use the
   family directly and maintainers can detect drift early
