# Crossing / Escape / Release Interface Map

This note governs the repository's handling of the `nissaraṇa / vimutti /
mutti / tarati / tiṇṇa / yogakkhema` interface.
It is not a general essay. It exists to stop release-side language from
collapsing into one vague English blur.

## Core Distinctions

| Term | Default | Functional Role | Do Not Collapse Into |
| --- | --- | --- | --- |
| `nissaraṇa` | `escape` | Directional way out from a danger, entanglement, or conditioned setup | generic `liberation`, `vimutti` |
| `vimutti` | `release` | Core release-family term | `nissaraṇa`, `mutti`, crossing language |
| `mutti` | `freedom` | Shorter result-noun in the release family | uncontrolled synonym of `vimutti` |
| `tarati` | `crosses over` | Action/process of crossing a danger such as a flood | generic `escape`, generic `release` |
| `tiṇṇa` | `crossed over` | Accomplished crossing or gone-beyond state | generic `liberated`, generic `freed` |
| `yogakkhema` | `security from the yoke` | Stabilized safety on the far side of the yoke family | generic `peace`, generic `safety` |

## Interface With Already-Stabilized Families

| Family | Default | Interface Role |
| --- | --- | --- |
| `ogha` | `flood` | What is crossed over by `tarati` or `tiṇṇa` |
| `yoga` | `yoke` | What `yogakkhema` names security from |
| `gantha` | `knot` | What stays distinct from release-side abstractions |
| `saṃyojana` | `fetter` | Binding framework distinct from crossing/escape language |
| `āsava` | `outflow` | Wears away or is exhausted; not itself crossing imagery |
| `vossagga` | `relinquishment` | Letting-go side of release, not identical with `vimutti` |
| `virāga` | `fading` | Cooling/fading side of release-sequence language |
| `nirodha` | `quenching` | Quenching side of culmination language |
| `khaya` | `wearing away` | Exhaustion side of culmination language |

## Operational Reading Order

1. If the text names the way out from something, prefer `nissaraṇa`.
2. If the text names release itself, prefer `vimutti`.
3. If the text uses the shorter noun and names freedom directly, prefer `mutti`.
4. If the text presents movement across a danger, prefer `tarati`.
5. If the text presents having already crossed, prefer `tiṇṇa`.
6. If the text names stable safety on the far side of the yoke family, prefer `yogakkhema`.

## Process and Result

| Process / Direction | Result / Condition |
| --- | --- |
| `nissaraṇa` | `vimutti` |
| `tarati` | `tiṇṇa` |
| `vossagga` | `mutti` |
| `yoga` | `yogakkhema` |

This is a translator-facing control aid, not a rigid universal sequence.
The texts can bring these terms close together without making them
interchangeable.

## Imagery Discipline

- Keep flood-crossing language audible when `ogha` or the act of crossing is in play.
- Keep yoke imagery audible when `yogakkhema` is in play.
- Let `nissaraṇa` keep its out-from directional force.
- Do not replace completed crossing with generic `liberation` language by habit.
- Do not let `vimutti` swallow every neighboring release-side term.

## Translator Guardrails

- Do not translate `nissaraṇa`, `vimutti`, and `mutti` all as `freedom`.
- Do not translate `tarati` and `tiṇṇa` as if they were merely synonyms for `vimutti`.
- Do not let `yogakkhema` lose its relation to the `yoga` family.
- Do not erase the release-side contrast with the bondage-imagery family:
  flood and crossing,
  yoke and security,
  knot and untied freedom are related images, not the same English word.

## Contrast Matrix

| If the text suggests... | Prefer | Avoid by Default |
| --- | --- | --- |
| a way out from danger or entanglement | `escape` | `release`, `freedom` |
| release itself as the governed house term | `release` | `escape`, `crossing over` |
| shorter result-noun naming freedom directly | `freedom` | automatic `release` |
| moving across a flood or danger | `crosses over` | `escapes`, `is liberated` |
| already on the far side | `crossed over` | `freed`, `liberated` |
| stable safety from the yoke family | `security from the yoke` | generic `peace`, generic `safety` |
