# SN 22.48 Translation Notes

This document is the companion rationale for
[SN 22.48: Khandha Sutta](sn22-48-khandha-sutta.md). The main translation is
the primary study surface; this file records debated translation choices,
source-audit calls, and edition notes that govern it.

## Source Base

- Primary source: the Pali text of SN 22.48 as segmented in SuttaCentral's
  Bilara data.
- Control surface: the repository's current term policy, especially `khandha`,
  `pañcakkhandhā`, `pañcupādānakkhandhā`, `rūpa`, `vedanā`, `saññā`,
  `saṅkhārā`, `viññāṇa`, `āsava`, `upādāna`, and the generated five-heaps
  control sheets.
- Working method: the discourse was governed as the repository's compact
  definition surface for the plain heaps and the clung-to heaps together.

## Governing Decisions

- `khandha` remains `heap`.
- `pañcakkhandhā` remains `five heaps`.
- `pañcupādānakkhandhā` remains `five clung-to heaps`, following the fixed
  whole-expression record rather than mechanically expanding `upādāna`.
- The five plain constituent heaps are rendered as `form heap`,
  `felt-experience heap`, `recognition heap`,
  `putting-together-activities heap`, and `knowing heap`.
- The five appropriated constituent heaps are rendered as `form clung-to heap`,
  `felt-experience clung-to heap`, `recognition clung-to heap`,
  `putting-together-activities clung-to heap`, and `knowing clung-to heap`.
- `sāsavaṃ upādāniyaṃ` is rendered `with outflows and fit to be taken
  personally`. This is document-local wording that keeps both `āsava` and
  `upādāna` visible without treating the phrase as a new headword default.
- The range formula `past, future, or present, internal or external, coarse or
  subtle, inferior or superior, far or near` follows the existing SN 22.59
  surface.

## Re-audit Calls

- The Pali abbreviates the middle members of each fivefold list. This study
  edition expands them because SN 22.48 is being used as a control passage for
  the whole heap family.
- `putting-together activities` is used in the heap sections, following the
  `saṅkhārā` context rule for canonical five-heaps enumeration.
- `with outflows and fit to be taken personally` is intentionally a little more
  explicit than the compact compound label `clung-to heap`. The line defines
  what makes the second fivefold set the clung-to version of the first.
- The main text does not add a release close, not-self refrain, or practice
  expansion. The root is a definition discourse, and the translation keeps it
  compact.

## Practice Clarifications

- SN 22.48 is useful because it prevents the plain five heaps and the five
  clung-to heaps from collapsing into one English expression.
- The five heaps name the fivefold experiential analysis.
- The five clung-to heaps name the same fivefold field where outflows are
  present and the field is fit to be taken personally.
- This surface supports SN 22.59 by stabilizing the plain heap enumeration and
  supports SN 22.89 by stabilizing the clung-to heap enumeration used in the
  identity-residue discussion.

## Editorial Presentation

- The main translation file uses two section headings so the distinction
  between the five heaps and the five clung-to heaps is visually immediate.
- Those headings are editorial aids, not claims that such headings stand in the
  root text itself.
- The repeated range formula is expanded for each member because this short
  surface is meant to function as a stable reference document.

## Edition Status

- This companion now supports the first stable study edition of the repo's
  SN 22.48 translation surface.
- Future revisions should be driven by source-audit gains, term-policy
  expansion, or real readability improvements, not routine wording churn.
