# Voice Standard

## Purpose

This document defines the repository's default sentence patterns for notes,
context rules, example notes, and contributor-facing prose.

Use it with:

- [../STYLE_GUIDE.md](../STYLE_GUIDE.md)
- [MODERN_ENGLISH_POLICY.md](MODERN_ENGLISH_POLICY.md)
- [term-entry-standard.md](term-entry-standard.md)

## Core Voice

Write like a careful translator explaining a governed decision to another
careful reader.

That means:

- modern
- direct
- precise
- calm
- readable in one pass

Avoid:

- archaic or pseudo-scriptural phrasing
- academic throat-clearing
- fragment-heavy placeholder prose
- casual or chatty wording

## Rule Statements

Prefer direct declarative or imperative sentences.

Use:

- `The project prefers ... because ...`
- `The project keeps ... because ...`
- `Translate this as ...`
- `Use this when ...`
- `Do not use this for ...`

Avoid:

- `It should be understood that ...`
- `It is to be noted that ...`
- `Project preference is ...`
- `is to be translated as ...`

## Context-Rule Notes

Use condition -> instruction wording.

Preferred patterns:

- `Use this by default.`
- `Use this as a controlled alternate when ...`
- `Use this when ...`
- `Do not use this for ...`
- `Keep the contrast with ... visible.`

Avoid fragment-only labels such as:

- `Default project rendering.`
- `Default rendering.`
- `Acceptable alternate.`
- `Use sparingly.`

## Explanatory Notes

Use short paragraphs that do real work.

Each note should answer one or more of these questions:

- Why is this the default?
- What confusion is this rule preventing?
- When does the default not apply?
- How does this term stay distinct from related terms?

Keep the prose direct. Do not add filler framing.

## Example Notes

Example notes should describe what the example shows.

Preferred patterns:

- `Shows the headword in compound use.`
- `Shows a practical contrast.`
- `Shows the formula in a canonical setting.`
- `Shows a source-facing line with the term left untranslated.`
- `Shows the standard family member in context.`
- `Shows the contrast line in a parallel form.`

Avoid fragment-only notes such as:

- `Useful compact analytical usage.`
- `Simple practical usage.`

## Example Sentences

Example translations and mini-illustrations should stay short, direct, and
easy to scan.

Preferred patterns:

- use a complete phrase or sentence rather than a label fragment
- keep the example focused on one distinction
- use parallel syntax in contrast pairs where possible
- keep explanation in the note line rather than loading it into the example

Examples:

- `with contact as condition, felt experience`
- `what one recognizes, one thinks about`
- `Use 'discernment' here. Do not switch to 'wisdom' just to sound elevated.`

Avoid:

- ornate slogan-like examples
- examples that explain three things at once
- contrast pairs with different sentence shapes

## Contributor-Facing Docs

Contributor docs should:

- tell the reader what to read
- tell the reader what to run
- tell the reader what standard to follow

Use short imperative or declarative lines.

Preferred patterns:

- `Read these documents in order.`
- `Run the full suite before opening a pull request.`
- `Use this audit after broad wording changes.`

Avoid indirect framing that delays the instruction.

## Editing Checklist

Before merging a wording change, check:

1. Does the note sound like direct house policy rather than a placeholder?
2. Does the context-rule note tell the reader what to do?
3. Does the example note describe what the example shows?
4. Does the entry sound like its related terms?
5. Does the contributor guidance use the same direct voice?
