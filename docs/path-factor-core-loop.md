# Path Factor Core Loop

This note governs one OSF-facing clarification already implicit in the repo and
now made explicit:

- `sammā-diṭṭhi`
- `sammā-sati`
- `sammā-vāyāma`

are not to be treated as static checklist items.

They are core live factors that work in constant exchange with one another and
help animate the rest of the noble eightfold path.

## Governance

- Keep the default renderings stable:
  `right view`, `right remembering`, `right effort`.
- Allow notes to explain these factors dynamically:
  right view looks and sees the trajectory,
  right remembering returns and keeps the task alive,
  right effort makes the wholesome change.
- Do not let this explanation collapse the factors into one another.
- Do not let it reduce `sammā-diṭṭhi` to belief, `sammā-sati` to mindfulness,
  or `sammā-vāyāma` to raw willpower.

## Practical Loop

- right view sees what is here and where it is heading
- right remembering keeps that task present
- right effort changes what should be changed
- then the same cycle can recur again

This explanation is allowed because it sharpens translator control for OSF
practice language without displacing the doctrinal path-factor framework.
